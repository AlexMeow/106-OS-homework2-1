🤔 功能是什麼？

  1. 將原本的排程 Round Robin 改成為 Priority Scheduling，並隨著使用及等待的次數自動加減每個 process 的 priority，使 process 不會因為 priority 過低而一直沒被執行。
  2. 除了自動加減外，使用者也可以自行更動其 priority ，並可以隨時查詢現有 process 的 id, priority, state 及使用時間。
  3. 關機功能。

😇 如何實作？

  • 實踐 Priority Scheduling 先要有 default priority (ex: 5)，在 scheduler() 中透過迴圈比較每個 process 的 priority 來找到最大 priority 的 process，然而在 swtch() 回來到 scheduler() 前將執行完的 process 的 priority 降低，並提升其餘的 process。
  • 建立 changePriority(pid, priorityValue) ，來更改 priority 並 return pid，若無找到此 pid 則 return -1。
  • 時間方面可透過 lapic.c 中的 cmostime() 並建立 date.c 來取得。
  • 建立 list() 顯示列表，並透過 lock 以防在顯示的過程中其狀態改變。
  • 關機則運用 outw() 指令來控制。

🐻 預期成效（怎麼用、誰用）

  • 使用者輸入 list 則可看見目前所有process的狀態 (pid, state, etc)

  • 使用者輸入 changeP <pid> <priority> 則將更新其 pid 的 priority

  • 使用者輸入 shutdown 則可離開 xv6
