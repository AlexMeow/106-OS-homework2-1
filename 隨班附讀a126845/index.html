🤔 功能是什麼？

xv6 中很可惜的是沒有實作與 Linux 中內建的 perf 的效能量測指令類似的 system call，因此想要來挑戰實作一個類似 perf 的 system call。下面簡單介紹 perf 這個指令。

Perf 簡介

從 Linux 2.6.31 以後，內建了 perf（全名為 Performance Event）系統效能分析工具（Profiler tool），透過 tracepoint 以及核心內部的特殊計數器（counter），分析執行中的程式碼，例如執行次數等統計結果，來理解運行中的程式效能好壞以及瓶頸所在。

指令的使用方法，可以在 cmd 中下 perf --help 指令來取得更多的使用方法說明。

參考資料

  • Linux 效能分析工具：Perf
  • 詳細使用方法可以參考這篇教學文
    • Tutorial - Linux kernel profiling with perf - Commands

而我們預計在 xv6 上實作出 perf list、perf top、perf stat 等三種指令。（時間允許將繼續實作 perf record / report 以及 perf sched 作為附加功能）

😇 如何實作？

Header File

因為這個功能與 process 有很大的關係，因為要記錄的就是 process 中執行的某些重要數據。

首先，在主要的 process 的 header file 中，也就是 proc.h 創造一個我們設計的 system call function 的指令，並定義好資料結構，例如











struct perf_data
{
    // process ID
    pid_t p_id;
    // counter
    int   cnt1; 
    float cnt2;
    // state: sleep/wake?
    bool  sleep;
};

Main Function

主要實作的部分則是確認 process 在執行 function call 的時候會經過哪些路徑，並在這些數個重要路徑上，設定 tracepoint，當 process 通過時， counter 就加1。程式碼的部分則是加在 proc.c 當中，在執行 Round-Robin 的排程演算法中，額外插入 tracepoint 以及 performance counter 程式碼。

當最終程序執行完之後，就能夠得到執行過程的數據，並且得以評估其表現，實作細節請見 Google 文件

🐻 預期成效（怎麼用、誰用）

給使用者使用，指令與 perf 相同，後面再加上 flag ，來切換使用的模式。

$ perf --help

 usage: perf [--help] [OPTIONS] COMMAND [ARGS]

 The most commonly used perf commands are:
   list            List all symbolic event types
   top             System profiling tool
   stat            Run a command and gather performance counter statistics

 See 'perf help COMMAND' for more information on a specific command.

perf sched（Bonus）

sched record 可以蒐集排程器相關數據 / sched latency 產生採樣結果，主要有下列項目

  • Task：name 與 pid
  • Runtime：實際運行時間
  • Switches：context switch 發生次數
  • Average delay：平均延遲時間
  • Maximum delay：最大延遲時間
$ perf sched -h

 Usage: perf sched [<options>] {record|latency}

$ perf sched record sleep 5
$ perf sched latency
